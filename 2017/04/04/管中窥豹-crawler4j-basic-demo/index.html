管中窥豹-crawler4j basic demo

发表于 2017-04-04

下载crawler4j源码之后，开始运行example/basic/BasicCrawlController进行测试。

位于basic目录下，仅有两个文件，查看源码可知，BasicCrawlController是负责爬虫控制规则的配置，而BasicCrawler可配置特定的网页访问规则以及对于采集的网页的操作。

BasicCrawler

首先，通过BasicCrawler探索爬虫的大概功能。

BasicCrawler继承WebCrawler，重写了shouldVisit和visit。前者是对于网页是否应该被采集进行相关改写，添加特定规则。此处跳过以图片后缀结尾的网页抓取。后者在完成抓取网页之后进行调用。

那么，爬虫是如何完成网页的爬取操作呢？

了解了BasicCrawler的作用，接下来查看WebCrawler源码。

可以看到，WebCrawler实现了Runnable接口，用于多线程中，调用其中的run方法。所以，我们重点关注run方法中的processPage(..)，这个方法就是爬虫处理网页的核心了。

简要概述，processPage(..)处理正在采集的url，进行网络请求，解析。并将网页信息存储在Page对象中。
其中黄色字块就是调用在BasicCrawler中重写的方法，而蓝色模块为理解processPage方法的核心，将在下面阐述。
